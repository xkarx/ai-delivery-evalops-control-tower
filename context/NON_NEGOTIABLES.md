# Non-Negotiables

1. Preserve the complete end-to-end lifecycle. EvalOps is central, not a substitute for product delivery.
2. Build visual, interactive, responsive dashboards.
3. Make V1 cloneable and reusable.
4. Generate structured, internally consistent synthetic data with shared IDs.
5. Agents must genuinely analyze evidence and brainstorm features.
6. Evals must execute. Never display invented scores as measured results.
7. Traces, tool calls, cost, latency, retries, and failures must be inspectable.
8. Require human approval at risky transitions.
9. GitHub engineering artifacts must be real when connected.
10. Demo mode must work without credentials.
11. Live adapters must activate when credentials are supplied.
12. Never commit or request secrets inside prompts.
13. Reuse open-source code only after license review and attribution.
14. Maintain requirement traceability and implementation status.
15. Clearly distinguish synthetic, simulated, generated, mocked, and live data.
16. The user should mainly perform account creation, OAuth, secret entry, workspace/repository selection, and final approvals.
17. Do not silently defer requirements. Record V2 deferrals with rationale.
18. Build a polished vertical slice, not disconnected mock modules.
